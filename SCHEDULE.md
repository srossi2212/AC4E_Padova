# Schedule

Five-hour Padova workshop sequence. Times are relative to the start of the live
session.

| Time | Block | Activity | Participant output |
| --- | --- | --- | --- |
| 00:00-00:15 | Orientation | Goals, repository map, tool lanes, research boundary | Tool lane chosen |
| 00:15-00:45 | Foundations | Agent loop, read/plan/act/verify, privacy boundaries | First scoped prompt |
| 00:45-01:20 | Card-Krueger scaffold | Project brief, data source, design memo, starter article | Draft brief or memo |
| 01:20-01:30 | Break | Setup reset | Working editor/terminal path |
| 01:30-02:15 | Harness pieces | Skills, subagents, hooks, optional MCP | Harness component selected |
| 02:15-03:00 | Orchestration | GitHub issues, branches, cloud agents, swarms, loops, goals | Small issue or goal prompt |
| 03:00-03:45 | Research sprint | Agent-assisted memo/code/paper task on Card-Krueger or own project | One changed artifact |
| 03:45-04:30 | Verification | Replication checker, referee reviewer, Playwright evidence | Review or verification log |
| 04:30-05:00 | Transfer | Checklist, adoption plan, next 7 days | Personal adoption plan |

## Demo Sequence

1. Open `examples/starter_article/`.
2. Read `AGENTS.md`, project brief, and research design memo.
3. Ask an agent to explain the Card-Krueger task without editing files.
4. Add or revise one scoped artifact.
5. Run or inspect one verification step.
6. Record what was delegated, what was verified, and what remains human-owned.

## Workshop Rhythm

- Short explanation.
- Bounded demo.
- Participant exercise.
- Verification or reflection.
- Commit/checkpoint before moving on.

## After The Workshop

Use the checklist in [`docs/research_article_harness.md`](docs/research_article_harness.md)
to apply the same harness to one real project. Start with one narrow issue; do
not ask an agent to reorganize a whole research repository in one prompt.
